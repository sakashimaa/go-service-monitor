## Команды Make

Полный список доступных команд:

```bash
make help
```

| Команда                | Описание                                            |
|------------------------|-----------------------------------------------------|
| `make build`           | Локальная сборка бинарника в `bin/`                 |
| `make docker-build`    | Сборка Docker-образа                                |
| `make clean`           | Очистка артефактов сборки (`bin/`)                  |
| `make run`             | Генерация Swagger-документации и запуск сервиса     |
| `make up`              | Сборка и запуск контейнеров                         |
| `make down`            | Остановка контейнеров (данные БД сохраняются)       |
| `make restart`         | Перезапуск контейнеров                              |
| `make deps`            | Загрузка и актуализация зависимостей                |
| `make fmt`             | Форматирование всего проекта                        |
| `make lint`            | Проверка линтером                                   |
| `make test`            | Запуск тестов                                       |
| `make test-verbose`    | Запуск тестов с подробным выводом                   |
| `make test-cover`      | Запуск тестов с выводом процента покрытия           | 
| `make test-cover-html` | Запуск тестов с выводом покрытия в html             |
| `make swag`            | Генерация Swagger-документации                      |
| `make migrate-up`      | Применение миграций БД                              |
| `make migrate-down`    | Откат миграций БД                                   |
| `make db-reset`        | Удаление данных БД и перезапуск контейнера postgres |
| `make logs`            | Просмотр логов контейнеров в реальном времени       |
| `make ps`              | Просмотр статуса запущенных контейнеров             |
| `make shell`           | Интерактивная консоль внутри контейнера приложения  |

## Локальная разработка

Загрузка зависимостей:

```bash
make deps
```

Сборка бинарника:

```bash
make build
```

Запуск (с автогенерацией Swagger):

```bash
make run
```

Форматирование кода:

```bash
make fmt
```

Проверка линтером:

```bash
make lint
```

Запуск тестов:

```bash
make test
```

Запуск с подробным выводом:

```bash
make test-verbose
```

Запуск с отображением покрытия кода тестами:

```bash
make test-cover
```

Генерация HTML отчета о покрытии с конкретными строками

```bash
make test-cover-html
```

После генерации отчёт можно открыть в браузере:

```bash
open coverage.html   # macOS
xdg-open coverage.html   # Linux
```

Запуск конкретного теста по имени:

```bash
go test -run TestCheckerRun ./internal/checker/...
```

## Управление данными (Docker Volumes)

База данных PostgreSQL использует Docker Volumes для персистентного хранения данных — они сохраняются при перезапусках и падениях контейнера.

Остановить проект (данные БД сохранятся):

```bash
make down
```

Перезапустить контейнеры (например, после правки `configs/sites.yaml`):

```bash
make restart
```

Полный сброс: удалить данные БД и поднять только postgres заново:

```bash
make db-reset
```

## Миграции базы данных

Миграции лежат в `migrations/` и управляются через [goose](https://github.com/pressly/goose).
Через `make` доступны три операции: создание новой миграции, накат и откат.

### Создание новой миграции

```bash
make migrate-create name=add_users_table
```

Это создаст в `migrations/` файл вида:

```
20260709120000_add_users_table.sql
```

Число в начале имени — timestamp создания, goose использует его, чтобы определить порядок применения миграций. **Нельзя переименовывать или редактировать существующие файлы миграций** после того, как они применены в любом окружении (включая локальное) — goose отслеживает применённые миграции по имени файла в служебной таблице `goose_db_version`, и ручное изменение уже применённой миграции не приведёт к повторному применению, а рассинхронизирует состояние базы с историей миграций.

Каждый файл миграции состоит из двух секций:

```sql
-- +goose Up
-- +goose StatementBegin
CREATE TABLE ...
-- +goose StatementEnd


-- +goose Down
-- +goose StatementBegin
DROP TABLE ...
-- +goose StatementEnd
```

`Up` — то, что применяется при `make migrate-up`, `Down` — то, что откатывается при `make migrate-down`. Заполнять `Down` обязательно, даже если кажется, что откатывать не понадобится — без него `migrate-down` для этой миграции просто не сработает.

### Применение миграций

```bash
make migrate-up      # применить все ещё не применённые миграции
make migrate-down     # откатить последнюю применённую миграцию
```

Обе команды берут `DATABASE_URL` из `.env` (см. `Makefile`, секция `-include .env` / `export`).
